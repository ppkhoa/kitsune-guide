    Sourcing and Verifying Content¶

    Goal: identify high-quality sources and verify content authenticity before encoding to ensure optimal results.

    What You'll Learn

    • Content sources: Official, public, and private source types and their trade-offs
    • Quality evaluation: Video, audio, and subtitle quality assessment criteria
    • Verification methods: Checksums, metadata analysis, and authenticity checks
    • Tools: MediaInfo, FFmpeg, and verification utilities
    • Best practices: Consistent sourcing workflows and common pitfalls to avoid

    1. Understanding Content Sources¶

    Official Sources¶

    Source Type Examples Quality
    Physical media Blu-ray discs, UHD discs Highest quality, unmodified
    Streaming services Netflix, Amazon, Disney+ High quality, may have DRM artifacts
    Digital purchases iTunes, Vudu, Google Play High quality, variable bitrates

    Why Official Sources Matter

    Official sources provide the highest-quality, unmodified video and audio, ensuring the best foundation for encoding. Your final encode can only be as good as your source.

    Public Sources¶

    Source Type Examples Risk Level
    Public torrents Popular torrent sites High
    File hosts Direct download sites High
    Re-uploads Secondary distribution Very High

    Public Source Risks

    • Lower overall quality
    • Tampered or mislabeled files
    • Re-encodes pretending to be originals
    • Potential malware in archives

    Private / Exclusive Sources¶

    Source Type Examples Reliability
    Private trackers Invite-only communities High
    Curated FTPs Vetted access servers High
    Scene releases Pre-database verified Medium-High

    Benefits of Private Sources

    • Higher curation standards
    • Verified releases with checksums
    • Consistent quality expectations
    • Community accountability

    2. Evaluating Source Quality¶

    Video Quality Checklist¶

    Criterion What to Check Red Flags
    Resolution Matches intended use (1080p, 2160p) Upscaled content, wrong AR
    Bitrate High bitrate for resolution Unusually low bitrate
    Codec H.264, H.265, AV1 Unknown or legacy codecs
    Aspect Ratio Correct and uncropped Stretched or improperly cropped
    Artifacts Clean, minimal compression Blocking, banding, macroblocking

    Audio Quality Checklist¶

    Criterion What to Check Red Flags
    Format DTS-HD MA, TrueHD, Atmos, FLAC Lossy when lossless expected
    Channels Correct layout (2.0, 5.1, 7.1) Missing channels, wrong config
    Sync Properly synchronized Drift, offset issues
    Bitrate Appropriate for format Unusually low for codec

    Subtitle Quality Checklist¶

    Criterion What to Check Red Flags
    Availability Required languages present Missing expected tracks
    Timing Synced with dialogue Offset or drift
    Translation Accurate and readable Machine translation, errors
    Formatting Proper styling (ASS/SSA) Broken tags, wrong fonts

    3. Verifying Content Authenticity¶

    File Verification Methods¶

    Method Tool Purpose
    MD5 checksum md5sum Quick integrity check
    SHA-1 checksum sha1sum Standard verification
    SHA-256 checksum sha256sum High-security verification
    Metadata analysis MediaInfo Technical specification review
    Stream inspection FFmpeg Deep stream-level analysis

    Checksum Verification

    # Compare file hash with known good value
md5sum filename.mkv
sha1sum filename.mkv

# Verify against provided checksum file
md5sum -c checksums.md5

    Source Verification¶

    Check Method Indicator
    Cross-reference specs Compare to NFO or database Specs should match exactly
    Bitrate analysis MediaInfo inspection Appropriate for resolution
    Encoding parameters x264/x265 settings in metadata Professional settings expected
    File size Compare to similar releases Unusual sizes are suspicious

    Signs of Tampered Content

    • Low bitrate for claimed resolution
    • Incorrect resolution or aspect ratio
    • Suspicious encoding parameters
    • Missing or modified metadata
    • File size doesn't match expected range

    5. Best Practices¶

    Sourcing Workflow¶

    Step Action Purpose
    1 Identify trusted sources Build reliable source list
    2 Verify before download Check specs match expectations
    3 Checksum verification Confirm file integrity
    4 MediaInfo inspection Validate technical specs
    5 Visual/audio spot-check Confirm quality matches specs

    Documentation Habit

    • Keep logs of content sources
    • Save verification notes (checksums, MediaInfo reports)
    • Track quality issues for future reference
    • Share findings with trusted communities

    Community Collaboration¶

    Practice Benefit
    Engage with encoding communities Access to vetted sources
    Share verification techniques Collective knowledge improvement
    Report fake or tampered content Protect community quality
    Contribute to databases Help others verify content

    6. Common Pitfalls¶

    Unverified Sources¶

    Risks of Unverified Content

    • Malware: Executable files or scripts in archives
    • Corrupted files: Incomplete or damaged content
    • Fake uploads: Mislabeled or tampered content
    • Re-encodes: Quality loss from unnecessary processing

    Prevention: Always scan files and verify checksums before processing.

    Detecting Fake Content¶

    Red Flag What It Indicates
    Incorrect file size Re-encode or fake
    Format mismatch Not original source
    Unusually low bitrate Heavy compression
    Cropped/stretched video Improper processing
    Missing metadata Stripped or modified

    Quick Verification Steps

    1. Run MediaInfo on the file
    2. Compare file hashes to known good values
    3. Inspect visually in VLC or MPC-HC
    4. Check audio sync and channel layout
    5. Verify subtitle timing and availability
