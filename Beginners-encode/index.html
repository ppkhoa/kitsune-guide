    StaxRip Video Encoding Guide¶

    Goal: encode video with StaxRip using optimized x264 settings, proper audio configuration, and professional workflows.

    What You'll Learn

    • Getting started: Installation and first launch setup
    • Source handling: Loading MKV, Blu-ray, and DVD sources
    • x264 settings: Recommended encoding parameters with explanations
    • Audio configuration: Bitrate and codec recommendations by resolution
    • Subtitles: Management, sync, and styling workflows
    • Filters: Cropping, denoising, debanding, and color correction
    • Output: Encoding, muxing, and quality verification

    1. Getting Started¶

    Installation¶

    Download StaxRip from the official releases page. Choose either the installer (automated setup) or portable version (no installation required).

    ➡️ https://github.com/staxrip/staxrip/releases

    The first time you launch StaxRip, it will download additional dependencies including ffmpeg and MKVToolNix. Allow these to install—they're required for encoding.

    First Launch¶

    StaxRip may prompt you to download additional tools or updates on first run. Accept these—they provide core encoding functionality. Default settings are fine for now; you'll adjust them as needed throughout this guide.

    2. Loading Source Files¶

    Basic Loading¶

    Add source files by dragging them directly onto the StaxRip interface, or use Source → Open File. StaxRip handles:

    • MKV, MP4, AVI containers
    • Blu-ray folder structures
    • DVD VIDEO_TS folders
    • Raw video streams

    Working with MKV Files¶

    MKV files load quickly without indexing. StaxRip automatically detects: - All video tracks (codec, resolution, bitrate) - Multiple audio tracks (languages, codecs, channel layouts) - Subtitle tracks (SRT, ASS, PGS)

    Select the tracks you want to keep in the Audio and Subtitles tabs. Use Preview to verify everything loads correctly. For batch processing, drag multiple MKVs into the window.

    Working with Blu-ray Sources¶

    Don't Load .m2ts Files Directly

    Blu-ray files cause indexing problems, bad seeking, filter issues, and audio desync.

    Instead, prepare Blu-ray content first:

    Option 1 - Remux with MakeMKV: Creates an MKV container preserving all original tracks. Recommended for most users.

    Option 2 - Demux with eac3to:
    Extracts individual video, audio, and subtitle streams. See the eac3to demux guide for detailed instructions.

    Working with DVD Sources¶

    DVDs use a VIDEO_TS folder structure containing .vob files and .ifo metadata. Use StaxRip or PGCEdit to identify the correct title (main movie vs. special features).

    StaxRip will automatically index DVD files when you load them. DVDs typically contain multiple audio tracks (AC3, DTS, MP2) and subtitle languages—select the ones you need in the appropriate tabs.

    Output Directory¶

    By default, StaxRip outputs to the same directory as the source file. To change this: File → Settings → Directories.

    Organize Your Projects

    Create separate folders for each project to keep source files, intermediate files, and final outputs organized.

    3. Video Encoding Settings (x264)¶

    These settings are based on the Silent Aperture x264 Guide and balance quality, compression efficiency, and compatibility.

    Recommended Base Configuration¶

    Copy and paste these settings as a starting point:

    --preset placebo
--level 41
--me umh
--merange 32
--rc-lookahead 250
--profile high
--crf 16
--deblock -3:-3
--qcomp 0.70
--aq-mode 3 --aq-strength 0.80
--psy-rd 1.00:0
--no-dct-decimate
--ref 16

    Setting Explanations¶

    Preset: --preset placebo
    Uses the slowest encoding speed for maximum compression efficiency. We'll override settings that waste time without improving quality.

    Level: --level 41
    For 1080p content. Use 5.1 or higher for 4K.

    Motion Estimation: --me umh --merange 32
    Balanced algorithm for predicting object movement between frames. The extended range is required for high-resolution content.

    Ratecontrol Lookahead: --rc-lookahead 250
    Analyzes 250 future frames before assigning bits. Reduces to ~60 on systems with limited memory.

    Profile: --profile high
    Standard 8-bit encoding. Use --profile high444 for 10-bit or 4:4:4 encodes.

    CRF (Constant Rate Factor): --crf 16
    Maintains constant perceptual quality by adjusting bitrate dynamically. Range: 15.0-20.0. Lower = higher quality. Adjust based on your quality target.

    Deblocking: --deblock -3:-3
    Reduces blockiness from compression. Avoid 0:0 except for animation. Only try -4:-4 for specific content after testing.

    Quantizer Curve Compression: --qcomp 0.70
    Balances bitrate between simple and complex scenes. Works with mbtree (enabled by default when rc-lookahead ≥ 40). Range: 0.70-0.85.

    Adaptive Quantization: --aq-mode 3 --aq-strength 0.80
    Mode 3 biases toward dark scenes. Dynamically assigns bits within each frame. Strength range: 0.60-0.85.

    Psychovisual Optimization: --psy-rd 1.00:0
    Enhances edge detail for perceived sharpness. Use --psy-rd 0.60:0 for animation.

    DCT Decimation: --no-dct-decimate
    Prevents dropping flat blocks that would reduce quality. Always disable.

    Reference Frames: --ref 16
    Number of frames used for motion prediction. x264 adjusts automatically if necessary.

    Additional Options¶

    10-bit encoding: 

    --output-depth 10

    4:4:4 color space: 

    --output-csp i444

    Zones (scene-specific settings): 

    --zones 100,200,crf=15

    Two-pass encoding (only when targeting a strict bitrate): 

    Pass 1: --pass 1 --bitrate 8500
Pass 2: --pass 2 --bitrate 8500

    Testing¶

    Always Test First

    Before encoding a full file:

    • Encode a 10-20 second test clip
    • Test sharp scenes, dark scenes, and high-motion scenes
    • Adjust CRF, deblock, or AQ settings based on results

    4. Audio Configuration¶

    Audio recommendations are based on source type, resolution, and channel layout.

    Resolution Source Type Recommendation
    4K (2160p) Lossless multi-channel (3.0-7.1) Passthrough (TrueHD, DTS-HD MA, LPCM, Atmos)
    4K (2160p) Lossless stereo/mono (1.0/2.0) FLAC at original bit depth (16/24-bit)
    1080p Lossless multi-channel (7.1) E-AC3 1280-1536 kbps
    1080p Lossless multi-channel (5.1) E-AC3 768-1024 kbps
    1080p Lossless stereo/mono (1.0/2.0) FLAC 16-bit
    720p Lossless multi-channel (3.0-7.1) AC3 640 kbps
    720p Lossless stereo/mono (1.0/2.0) AAC transparent bitrate
    All Lossy sources Passthrough

    Never Re-encode Lossy Audio

    Passthrough lossy sources. Re-encoding lossy→lossy causes significant quality loss.

    Secondary Audio Tracks¶

    Secondary tracks include commentaries, alternate languages, and isolated scores.

    Lossless secondary tracks:
    Encode to AAC. Exception: passthrough isolated scores.

    Lossy secondary tracks:
    Passthrough unless repair or sync is needed.

    Multi-channel secondary tracks:
    Follow the same rules as primary audio.

    Best Practices¶

    • Avoid lossy→lossy re-encoding
    • Use DEE (Dolby Encoding Engine) when available for Dolby formats
    • Convert LPCM stereo/mono to FLAC
    • Dither to 16-bit FLAC for stereo/mono
    • Use higher bitrates for dynamic action content
    • Use lower bitrates for quiet dialogue-heavy content

    Downmixing¶

    Only downmix when necessary.

    7.1 → 5.1: Acceptable for 1080p/720p encodes. Use DEE for clean downmixing.

    Avoid: Downmixing to stereo causes major quality loss.

    Audio Filters¶

    Audio Filters Can Degrade Quality

    Only use audio filters if you understand their purpose.

    • Normalize: Balances volume but may reduce dynamic range
    • Dynamic Range Compression: Makes quiet scenes louder—avoid unless needed

    When in doubt, leave audio filters disabled.

    Validation¶

    Before final encoding: - Use Preview to check audio/video sync - After encoding, play several scenes - Check for distortion - Verify channel mapping (stereo/5.1/7.1) - Ensure sync is perfect

    5. Subtitle Management¶

    Embedded Subtitles¶

    StaxRip automatically detects embedded subtitles: - SRT (text-based) - ASS/SSA (styled subtitles) - PGS (image-based Blu-ray subtitles) - VobSub (DVD)

    In the Subtitles tab, select which tracks to include, exclude, or convert (PGS → SRT via OCR). Disable unwanted commentary or duplicate language tracks.

    External Subtitles¶

    Drag and drop .srt, .ass, or .sub/.idx files directly into StaxRip, or use Subtitles → Add Subtitle. External subtitles work identically to embedded ones.

    Subtitle Synchronization¶

    Subtitle timing issues are common when using subtitles from different releases or mixing WEB-DL and Blu-ray sources.

    Sync workflow: 1. Preview video in StaxRip → Preview 2. Check line timestamps against dialogue 3. If out of sync, fix subtitles before encoding

    Recommended tools: - Subtitle Edit - Aegisub (for ASS styling)

    Aegisub version guidance

    • Typesetters: Use the arch1t3cht fork.
    • Non-typesetters (timing, basic edits, cleanup): Use Aegisub 3.4.0.

    See upstream bug report:
    arch1t3cht/Aegisub issue 177

    Typical fixes: - Apply global offset - Stretch/compress timings - Adjust scene-by-scene

    Re-import the corrected file into StaxRip after fixing.

    Multi-Language Subtitles¶

    For releases targeting multiple audiences: - Select only relevant languages - Set correct language tags for each track - Mark the primary language as Default

    Finalize language tags and flags in MKVToolNix after encoding.

    Subtitle Styling (ASS/SSA)¶

    Styled subtitles contain custom fonts, positioning, colors, margins, and sign translations (anime). Use Aegisub for editing:

    • Fix margin issues
    • Adjust font sizes
    • Correct line breaks
    • Remove unwanted karaoke or effects

    Save and reload into StaxRip after editing.

    Validation¶

    Before full encoding, do a short test: - Verify subtitles load correctly - Confirm sync - Test embedded fonts (ASS) - Check burn-in if using hard subtitles - Test playback in VLC, MPV, or Plex

    For soft subtitles, verify language tags, track order, default/forced flags, and test in multiple players.

    6. Aspect Ratio Corrections¶

    Improper aspect ratio causes stretched or squished images, incorrect black bars, and player-dependent distortion. This section ensures your output displays correctly.

    Understanding SAR, DAR, and PAR¶

    SAR (Sample Aspect Ratio): The shape of each pixel in the source file. Most modern content uses square pixels (1:1), but DVDs, some WEB sources, and older TV content may have non-square pixels.

    DAR (Display Aspect Ratio): The final aspect ratio the viewer sees. Common examples: 16:9 (1.78), 4:3 (1.33), cinematic formats (2.20, 2.35, 2.39, 2.40).

    PAR (Pixel Aspect Ratio): A multiplier connecting SAR and DAR: DAR = (Width / Height) × PAR

    If SAR is incorrect, DAR displays incorrectly.

    When Corrections Are Needed¶

    WEB sources: Some streaming services deliver content with incorrect or inconsistent SAR metadata.

    DVD sources: Almost always require correction due to NTSC/PAL non-square pixels.

    Some Blu-rays: Rarely, poorly mastered Blu-rays have 1-2px borders or incorrectly flagged DAR.

    Anime releases: Occasionally need cropping due to edge artifacts or variable aspect ratio sequences.

    Fixing Aspect Ratio¶

    Step 1 - Inspect the source:
    Use the Preview window to check circles, faces, and straight lines. If circles appear oval, DAR is incorrect.

    Step 2 - Apply corrections:
    In the Filters tab, use: - Resize filter: Corrects resolution to match intended DAR - Crop filter: Removes black borders and windowboxing

    Examples:

    Fixing incorrect SAR (DVD): 

    Resize(720, 480).SetDAR(16:9)

    Adjusting WEB-DL with stretched image: 

    SetDAR(1.85)

    Cropping vs. Scaling¶

    Cropping removes unwanted edges: - Black bars - Dirty pixels - Encoder artifacts - 1-2px letterbox noise

    Cropping rules: - YUV420 requires mod2 (crop by even numbers) - To crop 1px, convert to 4:4:4 or RGB (VapourSynth/AviSynth) - Avoid overcropping to prevent losing content

    Scaling adjusts resolution to correct DAR: - Smooths slightly warped video - Matches official resolutions (1920×804, 1920×800, 3840×1608)

    Use scaling when: - DAR is off by a decimal (e.g., 2.39 → 2.40) - Content needs exact theatrical aspect ratio

    Recommendation: Slight cropping removes artifacts cleanly without altering framing. Use scaling when the required correction is too large for cropping or you need pixel-perfect theatrical dimensions. Avoid excessive cropping and unnecessary upscaling.

    Verification¶

    Before encoding: - Preview multiple shots - Compare against official screenshots when available - Export a short test encode - Check playback in MPV, VLC, Plex, and TV apps

    Look for: - Correct geometry (round objects should be round) - No visible distortion - No remaining border artifacts

    7. Filters and Processing¶

    Filters clean, enhance, or adapt video for the best output. The goal is minimal modification unless the source requires it. A clean source needs fewer filters; noisy or banded sources need more.

    Filter Chain Overview¶

    Open the Filters tab in StaxRip. Common filters include:

    Filter Purpose When to Use
    Resize Adjusts resolution Downscaling, DAR fixes
    Crop Removes borders, noise, windowboxing Black bars, dirty edges
    Denoise Reduces noise or grain Noisy WEB/TV sources
    Deband Smooths color gradients Visible banding in skies/gradients
    Sharpen Adds perceived detail After denoising/debanding
    ColorMatrix BT.601 ↔ BT.709, level fixes Wrong color matrix flags
    Action How To
    Add filter Click [+] and choose from list
    Remove filter Click [x] next to a filter
    Edit filter Double-click a filter to adjust parameters

    Cropping¶

    Cropping removes black bars, encoder noise at edges, bad pixels, telecine borders, and letterboxing.

    Auto-crop: 

    Crop → AutoCrop

    Manual cropping guidelines: - Use mod2 values for YUV420 (crop evenly) - Avoid cropping artistic content - Slight cropping (1-2px) acceptable for cleaning noisy edges - For 1px crops, use 4:4:4 intermediate script

    Resolution Adjustment¶

    Resizing is used to: - Downscale (2160p → 1080p) - Conform to theatrical aspect ratios (1920×804, 1920×800, 3840×1608) - Smooth small DAR fixes that cropping alone can't handle

    Setting resolution: 

    Filters → Resize → Set target resolution

    Best practices: - Maintain aspect ratio - Avoid unnecessary upscaling - Use high-quality scaling kernels: Spline36, Lanczos, or Bicubic with neutral parameters

    Denoising¶

    Removes excessive noise or digital grain.

    Over-denoising Destroys Texture

    Aggressive denoising produces plastic-looking results. Use sparingly.

    Common denoisers:

    Denoiser Type Notes
    KNLMeansCL GPU Fast, good quality
    HQDN3D CPU Lightweight, general purpose
    BM3D CPU Highest quality, very slow
    FFT3DFilter CPU Good for grain patterns

    When to denoise: - TV/WEB sources with compression noise - Overly grainy sources - Low-light scenes

    When NOT to Denoise

    • Blu-rays mastered with intentional grain
    • High-quality cinematic content
    • Anime with light noise patterns (risk of smearing)

    Debanding¶

    Banding appears in gradients (sky, smoke, fog, walls). If visible: - Apply a deband filter - Avoid overly strong settings (can blur detail)

    Common filters:

    Filter Framework Notes
    f3kdb AviSynth/VapourSynth Most popular, configurable
    GradFun3 AviSynth Lightweight alternative
    placebo.deband VapourSynth Best quality, GPU-accelerated

    Starting values: 

    f3kdb(range=15, grainy=16, grainc=16)

    Sharpening¶

    Restores lost texture after denoising or debanding. Use lightly to avoid halos or ringing.

    Filter Type Notes
    LSFmod AviSynth Configurable, popular
    LSharpen AviSynth Lightweight
    CAS Both AMD Contrast Adaptive Sharpening, natural results

    Sharpening Best Practice

    Apply sharpening as the last filter in the chain, after denoising and debanding. Start with low strength values and increase gradually.

    Color Correction¶

    Incorrect colors occur due to wrong matrix flags (BT.601 vs BT.709), WEB master issues, or improper conversions.

    In StaxRip, use ColorMatrix: 

    ColorMatrix(mode="Rec.601->Rec.709")

    Or apply: - Levels - Tweak (hue/saturation/brightness) - HDR → SDR conversions (best handled in VapourSynth)

    Preview and Testing¶

    Always verify filter results: 

    Preview → Scrub through timeline

    Look for: - Overcropping - Artifacts - Crushed blacks/blown highlights - Broken gradients - Sharpening halos

    Before full encoding: 

    Encode 30-60 seconds as a test

    Verify filters behave correctly, aspect ratio is fixed, no unexpected artifacts, and bitrate/CRF matches expectations.

    Saving Presets¶

    For frequently encoded similar sources: 

    Tools → Settings → Filters → Save as Preset

    Examples: Anime preset, live-action WEB-DL preset, HDR→SDR preset, DVD restoration preset.

    8. Encoding and Output¶

    Save Your Project¶

    Before encoding: 

    File → Save Project As…

    This lets you reuse settings, quickly fix issues without rebuilding, and efficiently batch encode similar content. Use descriptive names like MovieName_1080p_x264_Project.srip.

    Always Save Before Encoding

    If encoding fails or you need to adjust settings, a saved project lets you resume without reconfiguring everything from scratch.

    Review the Queue¶

    StaxRip supports batch encoding. Check the Jobs window: - Is your main job listed? - Are all jobs configured correctly?

    Add multiple files: Configure → Add to Queue
    View queue: Jobs → Show Jobs

    Start Encoding¶

    Click Start. StaxRip displays: - Real-time encoding log - Estimated time - Encoding speed (fps) - x264/x265 CLI output

    Monitor for warnings or errors. Red text or stopped encodes require log review.

    System Performance¶

    Encoding is CPU-heavy and may use GPU resources depending on filters.

    During encoding: - Close unnecessary programs - Avoid gaming or heavy multitasking - Ensure good airflow and temperatures - Monitor Task Manager for CPU/GPU/RAM usage

    If StaxRip slows your system: 

    Task Manager → Details → staxrip.exe → Set Priority → Below Normal

    Encoding Time¶

    Time varies based on source (resolution, noise, motion complexity), settings (CRF vs 2-pass, preset, filters), and hardware (CPU model, RAM speed, SSD vs HDD).

    Example estimates for 2-hour 1080p encode:

    Preset Time
    Medium 45-90 min
    Slow 2-5 hours
    Slower 6-10 hours
    Placebo 12-48 hours

    Final Muxing¶

    Recommended: Use MKVToolNix for Final Muxing

    Do NOT mux in StaxRip. External muxing with MKVToolNix provides:

    • Manual verification of all elements
    • Precise control over metadata
    • Proper flags, languages, and ordering
    • Cleaner, more professional files
    • Easier debugging

    Workflow: 1. Encode video only in StaxRip 2. Open MKVToolNix GUI 3. Add encoded video, audio tracks, subtitle tracks, and chapters 4. Set flags (default, forced, language tags) 5. Click Start Muxing

    Test the final result in MPC-HC, mpv, or VLC.

    Error Handling¶

    If encoding fails:

    Issue Cause Solution
    Missing filters Plugin not installed Check StaxRip dependencies
    Incorrect color formats Wrong input/output format Verify source color space
    Unsupported audio/subtitle Format incompatibility Convert or passthrough
    Invalid crop/resize Odd numbers in 4:2:0 Use mod2 values
    DGDecNV errors GPU/license issues Check NVIDIA driver, license

    Debugging Strategy

    Test a 10-second clip to isolate issues before attempting a full encode.

    If the script breaks: 

    Video → Script → Edit Script

    Check for incorrect function names, wrong paths, or missing plugins.

    Save Templates¶

    To reuse encoding setups: 

    Tools → Settings → Templates → Save as Template

    Template Use Case
    x264 1080p live-action Standard movie/TV encodes
    x264 Anime Animation with adjusted psy-rd
    x265 4K HDR UHD HDR content
    x265 SDR conversion HDR to SDR workflow
    DVD restoration Upscaling and cleanup

    Appendix A: DGDecNV Setup (Advanced)¶

    DGDecNV is an advanced toolset for NVIDIA GPU users that improves decoding and indexing performance for supported video formats.

    What is DGDecNV?¶

    Component Purpose
    DGSource High-performance video source filter for VapourSynth and AviSynth
    DGIndexNV Creates .dgi index files for StaxRip

    ➡️ https://www.rationalqm.us/dgdecnv/dgdecnv.html

    Requirements¶

    Requirement Details
    NVIDIA GPU Supported model with NVDEC
    License Hardware-locked key (purchase required)
    Formats H.264, HEVC, MPEG-2 only

    Installation¶

    Step Action
    1 Download DGDecNV from the website (license purchase required)
    2 Place DGIndexNV.exe and associated files in a folder
    3 In StaxRip: Tools → Settings → Apps → DGIndexNV → Set path
    4 Set default: Tools → Settings → Preprocessing → DGSource as default

    Verification¶

    Check Expected Result
    Load compatible file File loads without errors
    Filters tab DGSource appears as source filter
    Indexing .dgi file created successfully

    Caveats¶

    Limitation Details
    Format support H.264, HEVC, MPEG-2 only
    Hardware Requires supported NVIDIA GPU
    Licensing Hardware-locked keys
    Updates May require reconfiguration

    Advanced Users Only

    Most users don't need DGDecNV. Only consider if you have an NVIDIA GPU and regularly encode H.264/HEVC content where indexing performance is a bottleneck.
