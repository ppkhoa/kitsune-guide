    Dolby Vision Encoding Guide¶

    Goal: extract, edit, sync, and reinject Dolby Vision RPU metadata for accurate tone mapping and DV compatibility.

    What You'll Learn

    • RPU extraction: Pull Dolby Vision metadata from source files
    • Frame alignment: Sync HDR10 and DV sources for accurate metadata
    • Metadata editing: Modify RPU via JSON for active area and profile conversion
    • Injection workflow: Reinject modified RPU into encoded video
    • Profile handling: Convert between DV profiles (5, 7, 8.1)

    Tools Required¶

    Tool Purpose Link
    dovi_tool Dolby Vision metadata manipulation ➡️ https://github.com/quietvoid/dovi_tool
    FFmpeg Video extraction and analysis ➡️ https://ffmpeg.org
    MKVToolNix Final muxing (MKV container) MKVToolNix Guide
    MP4Box Final muxing (MP4 container) ➡️ https://github.com/gpac/gpac

    1. Extracting the RPU Metadata¶

    Extract Dolby Vision RPU metadata from the source video:

    ffmpeg -i source.mkv -c:v copy -vbsf hevc_mp4toannexb -f hevc - | dovi_tool -m 0 extract-rpu -

    Apple Source Compatibility

    Some Apple sources may fail with -m 0. If extraction fails, remove the flag and retry: 

    ffmpeg -i source.mkv -c:v copy -vbsf hevc_mp4toannexb -f hevc - | dovi_tool extract-rpu -

    2. Syncing Video Files (Frame Alignment)¶

    Before editing RPUs, verify that HDR10 and DV sources match frame-for-frame.

    Comparison Workflow¶

    Step Action Tool
    1 Load both original and target video AviSynth+ or VapourSynth
    2 Compare frames side-by-side Frame stepping through both sources
    3 Identify mismatches Extra intros, black frames, missing frames
    4 Create sync JSON Define frame operations

    Sync JSON Example¶

    {
  "remove": ["0-39"],
  "duplicate": [
    {
      "source": 0,
      "offset": 39,
      "length": 10
    }
  ]
}
    Operation Purpose
    remove Delete frames (e.g., extra intro frames)
    duplicate Copy frames to fill gaps
    source Source frame index
    offset Starting position for duplication
    length Number of frames to duplicate

    3. Verifying HDR Grade Consistency¶

    Scenario Behavior
    Same provider streams Usually share identical HDR grading
    L6 metadata differences Normal between HDR10 and DV
    Profile 7 FEL HDR10 metadata must match RPU metadata

    Profile 7 FEL Requirement

    For Profile 7 FEL sources, HDR10 metadata must exactly match RPU metadata for accurate tone mapping. Mismatched metadata causes incorrect brightness and color reproduction.

    4. Setting the Active Area (Letterbox Bars)¶

    If the source has black bars, define the active picture area in JSON:

    {
  "active_area": {
    "crop": true,
    "presets": [
      {
        "id": 0,
        "left": 0,
        "right": 0,
        "top": 210,
        "bottom": 210
      }
    ],
    "edits": {
      "0-40": 0
    }
  }
}
    Parameter Description
    crop Enable active area cropping
    presets Define crop configurations
    left/right/top/bottom Pixel values for each edge
    edits Apply preset to frame ranges

    5. Profile Conversion (Convert RPU to Profile 8.1)¶

    Source Profile Conversion Mode Use Case
    Profile 7 (UHD BD) Mode 2 Blu-ray disc sources
    Profile 5 + HDR10 Mode 3 Streaming sources paired with HDR10

    Conversion JSON Example¶

    {
  "mode": 3
}

    Mode Selection

    • Mode 2: Use for UHD Blu-ray Profile 7 sources
    • Mode 3: Use for streaming Profile 5 sources that need HDR10 pairing

    6. Apply Metadata Edits¶

    Apply JSON edits to the extracted RPU:

    dovi_tool editor RPU.bin -j edits.json

    Output: RPU_modified.bin

    7. Verify Modified Metadata¶

    Check the modified metadata before encoding:

    dovi_tool info -s RPU_modified.bin

    Verification Checklist¶

    Parameter What to Check
    Mastering Display e.g., 0.0001 / 1000 nits
    MaxCLL / MaxFALL Peak brightness values
    Scene cuts Frame-accurate scene detection
    Active area Correct crop values applied
    Profile Correct target profile (8.1)

    8. Special Cases¶

    Profile 7 FEL¶

    Critical Requirement

    HDR10 metadata must match RPU metadata exactly. Mismatched metadata causes tone mapping errors.

    Profile 5 → Profile 8.1¶

    Step Action
    1 Use Mode 3 for conversion
    2 Ensure HDR10 & DV metadata align
    3 Verify with dovi_tool info

    9. Remuxing Dolby Vision¶

    Step 1: Extract HDR10 Base Layer¶

    ffmpeg -i source.mkv -map 0:v -c copy hdr10.hevc

    Step 2: Inject Modified RPU¶

    dovi_tool inject-rpu hdr10.hevc --rpu-in RPU_modified.bin -o dv_final.hevc

    Step 3: Mux Final File¶

    Container Tool Guide
    MKV MKVToolNix MKVToolNix Guide
    MP4 MP4Box ➡️ https://github.com/gpac/gpac

    Final Muxing

    Use MKVToolNix or MP4Box to combine the Dolby Vision video with audio and subtitles for the final release.

    Complete Workflow Summary¶

    Step Command/Action Output
    1 Extract RPU RPU.bin
    2 Sync frames Alignment JSON
    3 Edit metadata edits.json
    4 Apply edits RPU_modified.bin
    5 Verify metadata Confirmation
    6 Extract base layer hdr10.hevc
    7 Inject RPU dv_final.hevc
    8 Mux final file final.mkv
