    Making Positional Subtitles for Your REMUX¶

    Guide by xzin

    Introduction¶

    Some films include positioned subtitles — text placed near the character speaking rather than centered at the bottom. This greatly improves readability and clarity.

    However, when PGS subtitles are OCR’d into SRT format, all positional data is lost.

    This guide explains how to capture subtitle positions from PGS and generate a mostly accurate ASS (Advanced SubStation Alpha) subtitle file that keeps the original placement.

    Software Needed¶

    • bdsup2sub
      Used to extract images & XML from PGS subtitle streams
    • Subtitle Edit
      Used to create and refine the final ASS subtitle file
    • pgs_ass_positioner
      Python script used to transfer positional data into the ASS file
    • Python
      Required for running the above script

    Step 1 — Extract Positional XML Data from PGS¶

    1. Open your .pgs subtitle file in bdsup2sub
    2. Export the subtitle using XML/PNG output format
    3. This produces many PNG subtitle images and one XML containing X/Y positions

    Step 2 — Create Your ASS Subtitle File¶

    There are two approaches:

    Option A — OCR your PGS using Subtitle Edit¶

    • Open the PGS file
    • Use Subtitle Edit’s OCR
    • Save as ASS format

    Option B — Use a web-sourced SRT¶

    • Must have exactly the same number of subtitle entries as the PGS
    • Otherwise the positional mapping will break

    Step 3 — Pre‑Style the ASS File in Subtitle Edit¶

    Open your ASS file in Subtitle Edit:

    Go to:
    File → Advanced Sub Station Alpha styles

    Configure:

    • Spacing: 0.0
    • Angle: 0.0
    • Alignment:
    • Left‑aligned, vertically centered
    • (First column, second row — very important!)
    • Margins (L/R/V): 0 / 0 / 0

    ⚠️ Alignment is the most crucial setting.
    Incorrect alignment will mirror your subtitle placement to the opposite side of the screen.

    Optional Styling¶

    You may set:

    • Font
    • Color
    • Font size

    For accuracy, sample the original PGS subtitle color.
    Example: RGB (205,205,205).

    Save the ASS file before continuing.

    Step 4 — Run the Positioning Script¶

    Use the script from:

    ➡️ pgs_ass_positioner

    Inside the script, set:

    xml_path = "path/to/exported.xml"
ass_path = "path/to/your.ass"

    The script scans entries like:

    <Graphic Width="711" Height="145" X="573" Y="785">en_0001.png</Graphic>

    It extracts:

    • X coordinate
    • Y coordinate
    • Height
    • Width

    Then it inserts these into your ASS file’s override tags.

    Script features:¶

    • Offsets Y by half the subtitle image height for improved accuracy
    • Includes x_offset and y_offset for global fine‑tuning

    The script outputs a new ASS file in the same directory.

    Step 5 — Verify and Fine‑Tune¶

    Open the new ASS file in Subtitle Edit:

    1. Load your video preview
    2. Ensure subtitles appear in their intended positions
    3. Adjust x_offset and y_offset in the script if needed
    4. Manually edit any outliers

    💡 Even though the method is very accurate, it’s not 100%.
    Watch through your film to verify all placements.

    Subtitle Edit displays the exact coordinates for each subtitle, making manual fixes easy.

    Final Notes¶

    • This process preserves true positional subtitle accuracy from PGS → ASS
    • Perfect for REMUX, archival work, anime, and foreign films
    • Thanks to xzin (original guide author)
