    WEB-DL Handling Guide¶

    Goal: clean, standardize, and prepare WEB-DL files for tracker-friendly distribution using tool-agnostic techniques.

    WEB-DL releases are direct digital rips from streaming platforms. They're valued for clean source quality, but raw WEB-DLs (especially StreamFab outputs) often contain redundant metadata, attachments, and poorly labeled subtitles.

    What You'll Learn

    • Metadata: Remove rip dates, genre tags, and auto-titles
    • Attachments: Strip covers, logos, and embedded images
    • Subtitles: Proper Forced + Full English + SDH with correct flags
    • MediaInfo: Clean, consistent technical metadata
    • Consistency: Follow release group standards

    Quick Start¶

    1. Ingest: Load the WEB-DL into MKVToolNix
    2. Clean: Remove auto titles, tags, dates, and attachments
    3. Subtitles:
      • Create Full English by stripping SDH
      • Add Forced subtitles when needed
      • Apply correct flags and locale tags
    4. Verify: Check MediaInfo for clutter and correctness
    5. Finalize: Mux and prepare tracker metadata

    Workflow Consistency

    A consistent workflow improves speed, accuracy, and reviewer trust.

    Release Checklist¶

    Before Upload

    • [x] No auto-titles, dates, or unwanted tags
    • [x] All non-essential attachments removed
    • [x] Forced track added (if needed)
    • [x] Full English clean track present
    • [x] SDH track properly flagged (never default)
    • [x] Locale codes correct (en-US, etc.)
    • [x] Default flag correct based on audio language
    • [x] MediaInfo clean and uncluttered
    • [x] Naming consistent and provider-correct

    Conventions¶

    Terminology Used in This Guide
    Term Meaning
    SDH Subtitles for the Deaf and Hard of Hearing
    Forced Foreign-dialogue-only subtitles
    Locale codes Full xx-YY format (e.g., en-US, fr-CA)
    Provider tags Streaming-source labels (AMZN, NF, DSNP, ATVP, etc.)
    Tags No provider junk, genre, or description fields
    Attachments Always stripped for clean MKVs unless fonts
    Chapters Scene markers embedded in MKVs; keep unless broken
    Default flag Indicates which track players auto-select
    HI flag Marks SDH tracks for accessibility
    OCR Converts image-based subtitles (PGS) into text
    PGS Bitmap subtitle format requiring OCR
    SRT Standard text subtitle format
    VTT WebVTT subtitles (convert to SRT)
    ASS Styled subtitle format (requires fonts; used in anime)
