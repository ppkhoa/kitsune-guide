    Subtitle Handling¶

    Goal: ship Forced, Full English, and English (SDH) subtitles, accurately tagged, correctly region-coded, and ordered in a way all players and trackers interpret correctly.

    What You'll Configure

    • Track Order: Forced → Full English → SDH
    • Flags: Correct default, forced, and hearing-impaired settings
    • Locale Codes: Proper BCP-47 region tags (en-US, fr-CA)
    • Formats: Convert VTT to SRT for compatibility
    • SDH Stripping: Create clean Full English from SDH tracks

    1. Strip SDH with Subtitle Edit¶

    Use Subtitle Edit to generate a clean "Full English" subtitle from an SDH caption file.

    Steps:

    1. Open the SDH subtitle in Subtitle Edit
    2. Go to Tools → Remove text for hearing impaired
    3. Save as a new subtitle (e.g., Movie.Title - English.srt)

    This becomes your Full English subtitle.

    When to Strip SDH

    Amazon (AMZN) frequently provides only SDH subtitles. Always check and create a clean Full English track when needed no matter the streaming service used.

    2. Correct Subtitle Ordering & Flags¶

    All WEB-DL releases should follow this subtitle order and flag configuration:

    Track Order Summary¶

    Order Track Title Default Forced HI Flag
    1st Forced English Forced NO YES NO
    2nd Full English (empty) NO* NO NO
    3rd English SDH SDH NEVER NO YES

    * Full English becomes default = YES only when audio is NOT English

    1. Forced English¶

    Used when the audio is English but characters speak another language in certain scenes.

    Rules:

    • Language: en-US (or correct locale)
    • Title: Forced
    • Flags:
      • forced = YES
      • default = NO

    This track should appear first.

    2. Full English Subtitles¶

    This is the cleaned subtitle created by stripping SDH.

    Rules:

    • Language: en-US
    • Title: (empty) → Displays as "English"
    • Flags:
      • default = OFF
      • forced = OFF

    Exception: If audio is NOT English, then:

    • Full English becomes default = ON

    This track always appears second (unless no Forced track exists).

    3. English (SDH)¶

    Closed captions with sound cues, speaker IDs, and accessibility notes.

    Rules:

    • Language: en-US
    • Title: SDH
    • Flags:
      • hearing impaired = YES
      • default = NO (ALWAYS)
      • forced = NO

    SDH should never be default and must appear last.

    Common Mistake

    SDH should never be set as default. Many players will auto-select it over the clean English track, causing unwanted hearing-impaired annotations.

    3. Region and Language Codes¶

    MKVToolNix uses BCP-47 locale codes, not plain language labels.

    Why Locale Codes Matter

    Correct locale assignment improves player auto-selection, subtitle sorting, and metadata accuracy in Plex/Jellyfin/Kodi.

    Common Locale Codes¶

    Language Region Code
    English US en-US
    English UK en-GB
    English Australia en-AU
    French France fr-FR
    French Canada fr-CA
    Spanish Spain es-ES
    Spanish Latin America es-419
    Portuguese Brazil pt-BR
    Portuguese Portugal pt-PT

    Region code reference: ➡️ Country Code List (ISO 3166-1)

    4. How to Determine the Correct Locale¶

    Check the streaming provider¶

    Provider Reliability
    Netflix Usually accurate
    Disney+ Usually accurate
    HBO Max Usually accurate
    Amazon Often inconsistent

    Provider Reliability

    Amazon frequently mislabels subtitle regions. Always verify by checking spelling conventions in the actual subtitle content.

    Inspect spelling¶

    Spelling Region
    color, center, theater American (en-US)
    colour, centre, theatre British (en-GB)

    Consider regional language variants¶

    • Québec French → fr-CA
    • France French → fr-FR
    • Latin American Spanish → es-419
    • Spain Spanish → es-ES

    5. Player Auto-Selection Behavior¶

    Plex / Jellyfin¶

    • Automatically selects Forced for foreign dialogue
    • Selects Default subtitle if language matches user preference
    • SDH should NEVER be default unless explicitly intended (rare)

    MPV / Kodi¶

    • Reads the default flag
    • Uses locale matching (en-US vs en-GB)

    6. Special Cases¶

    Only SDH exists¶

    Amazon (AMZN) frequently does this.

    Solution: Strip SDH to create Full English using Subtitle Edit.

    Forced + Full English both provided¶

    Order must stay: Forced → Full → SDH

    Bitmap (PGS) subtitles¶

    Workflow: OCR → clean text → title correctly → mux back into MKV.

    PGS OCR Tools

    Use Subtitle Edit for bitmap-to-text conversion.

    7. WebVTT → SRT Conversion¶

    Many streaming services provide subtitles in WebVTT (.vtt) format. While MKV supports VTT, SRT is strongly preferred for WEB-DL releases.

    Always Convert to SRT

    Converting WebVTT to SRT before muxing prevents compatibility issues and timing drift in media players.

    Why SRT is preferred¶

    • Universal compatibility across Plex, Jellyfin, MPV, Kodi
    • Cleaner formatting
    • No CSS/style or positional quirks
    • Fewer muxing issues
    • Better behavior in syncing tools (alass, AutoSubSync)

    When WebVTT causes problems¶

    • Includes unwanted positioning / CSS blocks
    • Timing drift on long content
    • Apple/ATVP VTT files contain metadata cues
    • Some VTTs load with blank or duplicate lines

    How to convert VTT → SRT¶

    1. Open .vtt file in Subtitle Edit
    2. File → Save as → choose .srt
    3. Subtitle Edit removes CSS & fixes formatting
    4. Manually verify correctness

    8. Final Checks Before Mux¶

    Pre-Mux Checklist

    • Forced only when needed
    • Full English title empty (displays as "English")
    • SDH title = SDH
    • SDH default = NO (ALWAYS)
    • Full English default = ON only when audio is not English
    • Correct locale codes (en-US, fr-CA, etc.)
    • UTF-8 encoding
    • Correct track order (Forced → Full → SDH)
    • All flags verified

    Quick Reference¶

    Track Title Language Default Forced Hearing Impaired
    Forced Forced en-US NO YES NO
    Full English (empty) en-US NO* NO NO
    SDH SDH en-US NEVER NO YES

    * default = YES only when audio is NOT English

    Result¶

    You get a clean, predictable, professional subtitle layout:

    • Forced for foreign dialogue
    • Full English for normal viewing
    • SDH for accessibility
    • Correct flags & language codes
    • Works perfectly in Plex, Jellyfin, MPV, Kodi, and all major players
    • Matches tracker and group standards
