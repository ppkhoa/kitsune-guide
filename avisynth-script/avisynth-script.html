    Automatically Create B-Frame Comparison Screenshots (Up to 5 Sources)¶

    Original Guide by Stelks

    This guide explains how to automatically generate B-frame comparison screenshots for up to five different video sources using AvsPmod and the ComparisonGen script.

    This is extremely useful for comparing encodes frame-by-frame, or producing comp sets.

    Requirements¶

    1. AvsPmod¶

    Install AvsPmod by following this guide:
    🔗 AviSynth+ Guide

    2. Required Plugins¶

    You must place the following plugins inside your plugins64 folder in AviSynth+.

    Image reference:

    Download pack (contains all required plugins):
    🔗 https://mega.nz/file/tX9QiYpA#BCBKQjEO8Cz7WJwCAx1ceSouGC_md1bH35ZQMUN3PoA Password: boobies

    Step 1 — Save the Script¶

    Save the following as bframe_comp.avs (or any name you like), then open it in AvsPmod.

    # Max clips is 5
# Default config is for 2 clips. Uncomment lines as needed
# For short movies/tv shows modify .trim(0, 150000) to be lower than total frame count
# Use .trim to sync sources of different lengths
# .crop = crop your encode from the source
# Uncomment ConvertBits(10) if source is 8-bit but encode is 10-bit
# Change "Encode" under .subtitle to group name

s=LWLibavVideoSource("Source File Path").crop(0, 276, -0, -276).trim(0, 150000).LWLInfo().subtitle("Source", align=9) #ConvertBits(10)
e1=LWLibavVideoSource("Encode File Path").trim(0, 150000).LWLInfo().subtitle("Encode", align=9)
#e2=LWLibavVideoSource("Other Encode Path").trim(0, 140000).LWLInfo().subtitle("Other", align=9)
#e3=LWLibavVideoSource("Other Encode Path").trim(0, 140000).LWLInfo().subtitle("Other", align=9)
#e4=LWLibavVideoSource("Other Encode Path").trim(0, 140000).LWLInfo().subtitle("Other", align=9)

# Frame_type:
# 1 = I-frame
# 2 = P-frame
# 3 = B-frame
# 4 = PB frame

# n = number of screenshots to take
# Too high = Invalid Frame error
# Delete: C:\Users\XXXX\AppData\Local\Temp\_frames_for_save.txt & frames_file.txt if needed

# 2 clips
ComparisonGen([s, e1], n=12, frame_type=[3,3], check_type=true, image_file=["C:\Comps\%01d_1source.png", "C:\Comps\%01d_2Encode.png"])

# 3 clips
#ComparisonGen([s, e1, e2], n=12, frame_type=[3,3,3], check_type=true, image_file=["C:\Comps\%01d_1source.png", "C:\Comps\%01d_2Encode.png", "C:\Comps\%01d_3Encode.png"])

# 4 clips
#ComparisonGen([s, e1, e2, e3], n=12, frame_type=[3,3,3,3], check_type=true, image_file=["C:\Comps\%01d_1source.png", "C:\Comps\%01d_2Encode.png", "C:\Comps\%01d_3Encode.png", "C:\Comps\%01d_4Encode.png"])

# 5 clips
#ComparisonGen([s, e1, e2, e3, e4], n=12, frame_type=[3,3,3,3,3], check_type=true, image_file=["C:\Comps\%01d_1source.png", "C:\Comps\%01d_2Encode.png", "C:\Comps\%01d_3Encode.png", "C:\Comps\%01d_4Encode.png", "C:\Comps\%01d_5Encode.png"])

# Output format works with UNIT3D comp sorted uploads

    Step 2 — Create Output Folder¶

    AvsPmod cannot create folders.

    Create this path manually (or update the script):

    C:\Comps\

    All generated screenshots will go here.

    Step 3 — Example Output¶

    By default, B-frame comps are output like this:

    Ordering for 2-clip comparisons:

    Source → Encode → Source → Encode → ...

    For 3+ clips:

    Source → Encode1 → Encode2 → Source → Encode1 → Encode2 → ...

    Step 4 — Example of Script Filled Out¶

    Notes & Tips¶

    • Read the script comments before running.
    • If you abort the job early, delete: 
      C:\Users\<YOU>\AppData\Local\Temp\_frames_for_save.txt
C:\Users\<YOU>\AppData\Local\Temp\frames_file.txt
    • Reduce n= if comps fail.
    • Crop sources properly for alignment.
    • Works best when clips have matching frame positions.
