    Audio Syncing Guide — WEB-DL, Crunchyroll, Amazon, and More¶

    Properly syncing audio to video is one of the most common tasks in WEB-DL work, especially when pairing higher‑quality audio from one provider (Amazon, iTunes, etc.) with better video from another (Crunchyroll, Disney+, Netflix).
    This guide explains multiple methods — from simple offsets to waveform matching.

    1. Why Audio Sync Issues Happen¶

    Different streaming platforms may:

    • Start streams at different timestamps
    • Add/remove silent frames, bumpers, or intro cards
    • Encode audio pipelines differently
    • Use varied chapters or pre‑roll segments

    This creates consistent offsets such as:

    • Audio starting too early
    • Audio starting too late
    • Rarely: drifting (length mismatch)

    If both audio tracks are the same length, you almost always only need a constant offset fix.

    2. Syncing Audio Using MKVToolNix (Easiest Method)¶

    MKVToolNix (MKVmerge GUI) allows you to mux video/audio and apply delay offsets.

    What MKVToolNix Is Good For¶

    • Perfect for simple offsets
    • Non‑destructive (no re‑encoding)
    • Extremely fast
    • Ideal when audio lengths match

    How To Apply an Audio Offset¶

    1. Open MKVToolNix GUI
    2. Add video (e.g., Crunchyroll)
    3. Add audio (e.g., Amazon DDP)
    4. Select the audio track
    5. In Properties → Delay (ms) enter your offset
    6. Mux the file

    3. Syncing by Comparing Waveforms (Accurate Method)¶

    For precise alignment, compare the waveforms of:

    • Crunchyroll AAC
    • Amazon DDP

    Tools for Waveform Comparison¶

    • Aegisub — subtitle‑aware waveform alignment
    • Audacity — simple waveform comparison
    • Resolve / Premiere — full timeline editors

    How to Compare Audio Tracks¶

    1. Extract both audio tracks
    2. Import into Aegisub/Audacity
    3. Zoom into the first speech or distinct sound
    4. Measure the difference
    5. Apply that offset using MKVToolNix, eac3to, or FFmpeg

    4. Syncing Audio Using eac3to (Lossless Delay Adjustment)¶

    eac3to applies audio delays without re‑encoding, ideal for DDP/EAC3/AC3.

    Why Use eac3to?¶

    • Completely lossless delay shifting
    • Fast and stable
    • Works well for WEB‑DL audio corrections
    • Produces a corrected standalone audio file

    Basic eac3to Delay Examples¶

    Apply a +250 ms delay (audio plays later):

    eac3to input.ddp output_fixed.ddp +250ms

    Apply a –250 ms delay (audio plays earlier):

    eac3to input.ddp output_fixed.ddp -250ms

    After Applying Delay¶

    1. Open MKVToolNix
    2. Add the corrected audio
    3. Mux normally — no additional delay required

    5. Syncing Audio Using FFmpeg (Command‑Line Method)¶

    FFmpeg can apply offsets while muxing video + audio together.

    Basic FFmpeg Sync Example¶

    ffmpeg -i crunchyroll.mkv -itsoffset 0.250 -i amazon_audio.ddp -map 0:v -map 1:a -c:v copy -c:a copy output.mkv

    This delays Amazon audio by 250 ms.

    FFmpeg Notes¶

    • Place -itsoffset before the audio input
    • Use -c copy to avoid re‑encoding
    • Works for all provider pairs:
    • Amazon audio + Disney+ video
    • Amazon audio + Crunchyroll video
    • iTunes audio + Netflix video

    6. Aegisub for Subtitle + Audio Syncing¶

    If subtitles are being edited:

    • Aegisub lets you sync subs to any audio track
    • Waveform + spectrogram views help identify offsets
    • Commonly used when Amazon audio is synced to Crunchyroll video

    Use when your workflow involves subtitle correction or timing.

    7. Automated Sync Detection Using OffsetFinder¶

    OffsetFinder:
    👉 https://github.com/c22dev/OffsetFinder

    OffsetFinder automatically:

    • Compares two audio tracks
    • Detects exact offset in milliseconds
    • Removes guesswork

    Perfect when:

    • Intros contain silence
    • Waveforms differ slightly
    • You want confirmation of your measurement

    Additional Automated Tool — BBC Audio Offset Finder¶

    BBC Audio Offset Finder:
    👉 https://github.com/bbc/audio-offset-finder

    A second automated tool that detects the alignment of one audio stream inside another.

    Pros:

    • Good for constant offsets
    • Useful when streams are similar but not identical

    Limitations:

    • Does not account for black frames
    • Does not fix different cuts
    • Only locates best waveform alignment

    Use this alongside OffsetFinder for double‑verification.

    8. Practical Workflow (Recommended)¶

    Step 1 — Extract Audio¶

    Use:

    • MKVToolNix
    • FFmpeg

    Step 2 — Compare Audio¶

    Using:

    • Aegisub
    • Audacity
    • OffsetFinder

    Determine the required offset (e.g., +250 ms).

    Step 3 — Apply Offset¶

    Using MKVToolNix:

    • Set audio delay to 250 ms in the properties panel

    Using eac3to:

    eac3to input.ddp output_fixed.ddp +250ms

    (FFmpeg example can be added here as needed.)

    Step 4 — Test Playback¶

    Use:

    • MPV
    • MPC‑HC
    • VLC

    Check:

    • Lip sync
    • Speech accuracy
    • OP/ED alignment

    Step 5 — Finalize Release¶

    If sync is correct, mux into your final WEB‑DL structure.

    9. Tips & Best Practices¶

    If the audio lengths match, it's almost always a simple offset.

    Most WEB-DL series will have the same offset for all episodes in the same season but you will still need to verify.

    If audio drifts over time:

    Possible causes:

    • Frame‑rate mismatch
    • Provider‑specific edits
    • PAL/NTSC conversions (rare)

    Great Provider Pairings

    • Amazon DDP + Disney+ video
    • Amazon DDP + Crunchyroll video
