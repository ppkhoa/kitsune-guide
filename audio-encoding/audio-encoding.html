    Audio Encoding Overview¶

    This guide outlines correct handling, conversion rules, and codec expectations for audio in encodes across SD, 720p, 1080p, and 2160p releases.
    It reflects standard tracker rules and best practices.

    1. List of Lossless and Lossy Codecs¶

    Lossless¶

    • DTS-HD MA
    • TrueHD
    • LPCM
    • FLAC

    Lossy¶

    • AC-3 / DD
    • DTS
    • DTS-HD HR
    • AAC
    • E-AC-3 / DDP / DD+
    • Vorbis
    • Opus

    2. Acceptable Conversions¶

    Encodes¶

    Main Audio¶

    • Lossless → Lossy allowed
    • Lossy → Lossy NOT allowed
    • Lossless only allowed for FLAC 1.0 / 2.0
    • Exception: Concerts & UHD encodes, where all lossless formats except LPCM are allowed.

    Secondary Audio (commentaries, isolated scores)¶

    • Lossless → Lossy allowed
    • Lossy → Lossy allowed
    • Lossless allowed only for isolated scores, not for commentaries.

    Remuxes¶

    Main Audio¶

    • Lossless → Lossy NOT allowed
    • Lossy → Lossy NOT allowed
    • Lossless → Lossless allowed in select cases
    • Typically for converting LPCM → FLAC, or remapping 1.0/2.0 lossless to FLAC.
    • DTS-HD HR must NOT be converted

    Secondary Audio¶

    • Lossless → Lossy NOT allowed
    • Lossy → Lossy NOT allowed
    • Lossless → Lossless allowed in select cases
    • Typically for 1.0 / 2.0 → FLAC

    3. SD Encodes¶

    Main Audio¶

    From Lossless 3.0 – 7.1¶

    • Preferred: 448 kbps AC3
    • Maximum: 640 kbps AC3
    • Note: AC3 does not support >5.1 channels

    From Lossless 1.0 / 2.0¶

    • Preferred: High-quality AAC
    • Maximum: 16-bit FLAC

    From Lossy¶

    • Leave untouched (passthrough)
    • Exception: DTS-HD HR MUST be transcoded to AC3

    Secondary Audio¶

    From Lossless¶

    • Medium-quality AAC / Opus

    From Lossy¶

    • Preferred: Medium-quality AAC / Opus
    • Maximum: Leave untouched (passthrough)

    All secondary audio rules assume 1.0/2.0.
    If higher channels exist, treat as main audio.

    4. 720p Encodes¶

    Main Audio¶

    From Lossless 3.0 – 7.1¶

    • Preferred: 640 kbps AC3
    • Maximum: 1509 kbps DTS
    • Note: AC3 does not support >5.1

    From Lossless 1.0 / 2.0¶

    • Preferred: High-quality AAC
    • Maximum: 24-bit FLAC (restricted bitrate)

    From Lossy¶

    • Leave untouched (passthrough)
    • Exception: DTS-HD HR → AC3 or DTS core

    Secondary Audio¶

    From Lossless¶

    • Preferred: Medium-quality AAC / Opus
    • Maximum: High-quality AAC / Opus

    From Lossy¶

    • Preferred: High-quality AAC / Opus
    • Maximum: Leave untouched

    5. 1080p Encodes¶

    Main Audio¶

    From Lossless 3.0 – 7.1¶

    • Preferred:
    • 1280–1536 kbps E-AC3 for 7.1
    • 768–1024 kbps E-AC3 for 5.1
    • Other acceptable: 640 kbps AC3, DTS up to 1509 kbps

    From Lossless 1.0 / 2.0¶

    • Preferred: 16-bit FLAC
    • Maximum: 24-bit FLAC (restricted bitrate)

    From Lossy¶

    • Leave untouched (passthrough)
    • Exception: DTS-HD HR → (E)AC3 or DTS core.

    Secondary Audio¶

    From Lossless¶

    • Preferred: Medium-quality AAC / Opus
    • Maximum: High-quality AAC / Opus

    From Lossy¶

    • Preferred: High-quality AAC / Opus
    • Maximum: Leave untouched

    6. 2160p Encodes (UHD)¶

    Main Audio¶

    From Lossless 3.0 – 7.1¶

    • Leave untouched
    • Do NOT convert to any lossy codec

    From Lossless 1.0 / 2.0¶

    • Preferred: FLAC (original bit-depth)
    • Maximum: Leave untouched
    • Exception: LPCM must be converted

    From Lossy¶

    • Leave untouched (passthrough)

    Secondary Audio¶

    From Lossless¶

    • High-quality AAC / Opus

    From Lossy¶

    • Leave untouched

    7. Miscellaneous Notes¶

    Programs to Encode AC3¶

    • Dolby Encoding Engine (DEE)
    • Audition 2017
    • Sound Forge Pro
    • Dolby Media Producer
    • FFmpeg & eac3to (less preferred; FFmpeg > eac3to)

    Programs to Encode E-AC3¶

    • DEE
    • Dolby Media Producer
    • Audition 2017
    • FFmpeg (less preferred)
    • For 7.1 E-AC3: must use Blu-ray profile (DEE/DMP only)

    Programs to Encode AAC¶

    • qaac (recommended)

    Additional Rules & Notes¶

    • LPCM must never be used in encodes or remuxes
    • LPCM 1.0/2.0 → FLAC

    • LPCM > 2.0 → FLAC / DTS-HD MA / TrueHD

    • TrueHD audio must include the AC3 core for compatibility

    • Lossy → Lossy transcoding allowed only in exceptional cases, such as:

    • No other source exists
    • Syncing audio from another source
    • Framerate restoration

    • Fixing major playback issues

    • Non-standard formats like >2.0 Opus/AAC allowed only for:

    • 1080p HDR encodes
    • 10-bit x264 animation encodes
