    Lossless Cutting & Merging Episodes (TVDB Ordering)¶

    Streaming platforms sometimes split or combine episodes differently than TVDB.
    To correct ordering without degrading quality, you may need to:

    • Remove duplicate intros
    • Remove ending credits
    • Merge two parts into one episode

    This guide shows how to do this losslessly, preserving WEB-DL status.

    ⚠️ Critical Rules¶

    Do NOT Re-Encode

    Any re-encode (even CRF 0) converts the file into a WEBRip.

    What Is Allowed

    • Lossless cutting with MKVToolNix
    • Timestamp cuts (nearest I-frame)
    • Appending files from the same source
    • Rebuilding with mkvmerge (no encoding)

    When This Is Necessary¶

    Common scenarios: - A service splits one TVDB episode into two - Duplicate credits appear mid-episode after merging - Intros or credits are inserted between parts

    If two episodes are being merged, credits from the first part should be removed.

    Method 1: Cutting by Timestamp (Recommended)¶

    MKVToolNix allows lossless timestamp-based cuts.

    Although not frame-accurate, MKVToolNix always snaps cuts to the nearest I-frame, ensuring the stream remains valid.

    Why Timestamp Cuts Are Safe¶

    • No decoding or re-encoding
    • Cut snaps to nearest I-frame automatically
    • Minor black frames are acceptable

    Cutting Credits by Timestamp¶

    1. Open the file in MKVToolNix GUI
    2. Go to Output
    3. Enable Split mode
    4. Select After output duration
    5. Enter the timestamp just before credits begin

    Method 2: Appending Episodes (GUI)¶

    Once credits are removed, you can append the next part using the MKVToolNix GUI.

    Requirements¶

    • Same source
    • Same codec, resolution, framerate
    • Same audio layout

    Appending in MKVToolNix¶

    1. Load the base episode
    2. Click Append
    3. Select the second file
    4. Verify stream alignment

    Result: Merged Episode¶

    At this point, the episode follows TVDB order and contains no duplicate credits.

    Method 3: mkvmerge Flow (CLI / Scripted)¶

    For automation, scripting, or headless systems, the same process can be done with mkvmerge.

    This is lossless and equivalent to the GUI append workflow.

    When to Use mkvmerge

    • Batch processing
    • Automation scripts
    • SSH / headless systems
    • Full control over track order and flags

    Step 1: Cut Credits (Optional)¶

    If you already created a cut file via GUI, skip this step.

    Example using timestamps (nearest I-frame is chosen automatically):

    mkvmerge -o part1_cut.mkv   --split timecodes:00:42:13.000   original_part1.mkv

    Step 2: Append Parts¶

    mkvmerge -o merged_episode.mkv   part1_cut.mkv + part2.mkv

    WEB-DL Safe

    Using mkvmerge this way preserves the original bitstreams.

    Method 4: Verifying I-Frames (Advanced / Accuracy Checks)¶

    If you want maximum accuracy when choosing cut points, you can inspect I-frame positions before making any cuts.

    This is optional, but useful when: - Cuts must land perfectly on scene changes - Audio transitions are sensitive - You want to minimize extra black frames

    VapourSynth (Recommended)¶

    VapourSynth allows frame-accurate inspection and easy identification of I-frames.

    ➡️ VapourSynth Guide

    Typical workflow: 1. Load the source into VapourSynth 2. Enable frame-type overlays 3. Identify nearest I-frame to your intended cut 4. Note the timestamp 5. Perform the cut using MKVToolNix or mkvmerge

    Tip

    VapourSynth is ideal for modern workflows and integrates cleanly with preview players like mpv.

    AviSynth (Legacy Option)¶

    AviSynth can also be used to inspect GOP structure and frame types.

    ➡️ AviSynth Guide

    Note

    AviSynth is Windows-only and more limited, but still effective for quick checks.

    Important Clarification¶

    You Still Cut With MKVToolNix

    VapourSynth and AviSynth are analysis tools only. The actual cutting should still be done with MKVToolNix or mkvmerge to remain lossless.

    Audio Considerations¶

    • Silent transitions are ideal
    • Dialogue near cut points should be checked
    • Minor silence gaps are acceptable

    Final Checklist¶

    Before Uploading

    • No re-encoding performed
    • No duplicate intros or credits
    • Episode follows TVDB order
    • MediaInfo still reports WEB-DL

    Summary¶

    Task Recommended Method
    Remove credits Timestamp cut
    Remove intro Timestamp > chapters
    Merge episodes (GUI) Append
    Merge episodes (CLI) mkvmerge
    Verify I-frames VapourSynth / AviSynth
    Preserve WEB-DL No re-encode
