    Dolby Vision Remuxing Guide¶

    Goal: verify and fix Dolby Vision remuxes using DGDemux and dovi_tool to ensure accurate metadata and playback.

    What You'll Learn

    • MakeMKV limitations: Common problems with Dolby Vision remuxes
    • DGDemux workflow: Superior tool for DV metadata handling
    • RPU verification: Compare and validate metadata accuracy
    • Fix procedures: Correct incomplete or misaligned RPU data
    • FEL handling: Special cases for Full Enhancement Layer discs

    Why DGDemux Over MakeMKV¶

    MakeMKV Limitations¶

    Problem Description Impact
    Mismatched scene cuts Incorrect BL/EL metadata merge Playback issues, tone mapping errors
    Incomplete RPUs Missing or misaligned metadata Visual artifacts, brightness problems
    Authoring sensitivity Struggles with flawed disc authoring GER editions particularly problematic

    Regional Disc Issues

    Retail UHD Blu-rays from certain regions (particularly German editions) often have flawed authoring that exacerbates MakeMKV's limitations. Always verify remuxes from these sources.

    DGDemux Advantages¶

    Feature Benefit
    Integrated dovi_tool Precise Dolby Vision metadata handling
    Proper layer separation Accurate BL/EL extraction
    Scene-cut preservation Maintains authoring intent
    Metadata verification Built-in validation capabilities

    Tools Required¶

    Tool Purpose Link
    DGDemux Primary demuxing tool for DV ➡️ https://www.rationalqm.us/dgdemux/binaries/
    dovi_tool RPU manipulation and verification ➡️ https://github.com/quietvoid/dovi_tool
    FFmpeg Fast extraction method ➡️ https://ffmpeg.org
    MKVToolNix Track extraction and muxing MKVToolNix Guide

    Verification and Correction Workflow¶

    Step 1: Demux the Enhancement Layer¶

    Extract the EL.hevc stream using DGDemux, then extract the RPU:

    dovi_tool extract-rpu EL.hevc -o RPU_EL.bin

    Step 2: Export Scene-Cut Data¶

    Extract scene-cut information for comparison:

    dovi_tool export -i RPU_EL.bin -d scenes

    Step 3: Extract RPU from Remux¶

    Compare the RPU from the remuxed file with the original.

    ffmpeg -i REMUX_P7.mkv -c:v copy -vbsf hevc_mp4toannexb -f hevc - | dovi_tool extract-rpu - -o RPU_P7.bin

    mkvextract tracks REMUX_P7.mkv 0:REMUX_P7.hevc
dovi_tool extract-rpu REMUX_P7.hevc -o RPU_P7.bin

    Step 4: Compare RPU Data¶

    Compare original and remuxed RPU files:

    diff RPU_EL.bin RPU_P7.bin
    Result Meaning Action
    No output Files match Remux is correct
    Differences shown Metadata mismatch Investigation required

    Profile 7 Verification

    For Profile 7 remuxes, always compare the RPU before and after injection/merge to ensure they are identical. This catches issues with mismatched frame orders or other anomalies.

    Step 5: Fix Incomplete RPUs (MEL Discs)¶

    For discs with Metadata Enhancement Layers (MEL), inject the correct RPU:

    Convert RPU from Profile 7 to Profile 8:

    dovi_tool -m 2 extract-rpu EL.hevc -o RPU_EL_P8.bin

    Inject corrected RPU into Base Layer:

    dovi_tool inject-rpu -i BL.hevc --rpu-in RPU_EL_P8.bin -o FIXED_REMUX.hevc

    Step 6: Handle FEL Discs¶

    FEL Limitation

    If the disc uses Full Enhancement Layer (FEL) metadata and brightness expansion is incorrect, no fix is possible. Wait for:

    • A corrected release from another region
    • A replacement disc with proper authoring
    Layer Type Fixable Method
    MEL (Metadata Enhancement Layer) Yes Profile conversion + injection
    FEL (Full Enhancement Layer) No Requires different source

    Complete Workflow Summary¶

    Step Action Command/Output
    1 Demux EL with DGDemux EL.hevc
    2 Extract original RPU RPU_EL.bin
    3 Export scene cuts Scene data files
    4 Extract remux RPU RPU_P7.bin
    5 Compare RPUs diff comparison
    6 Fix if needed FIXED_REMUX.hevc
    7 Mux final file Complete remux

    Troubleshooting¶

    Common Issues¶

    Issue Cause Solution
    RPU mismatch Authoring errors Re-demux with DGDemux
    Scene cut errors MakeMKV merge issues Use DGDemux instead
    Brightness problems FEL metadata issues Check if fixable (MEL vs FEL)
    Frame order issues Flawed disc authoring Verify with original EL RPU

    Regional Considerations¶

    Region Risk Level Recommendation
    GER High Always verify with DGDemux
    USA Low-Medium Spot check remuxes
    UK Low-Medium Spot check remuxes
    Other Variable Verify first remux from new source
