    SubExtractor OCR Guide¶

    OCRing VobSub (IDX/SUB) or DVD Subtitles into SRT¶

    This guide covers how to use SubExtractor to OCR image-based subtitles (VobSub / IDX+SUB or DVD folders) into editable SRT files.

    Tools You'll Need¶

    • SubExtractor
    • Subtitle Edit
    • Inviska MKV Extract
      or gMKVExtractGUI
    • DGIndex (required only for DVD folder sources)

    Getting Started¶

    SubExtractor can process: - DVD folders (VIDEO_TS) - VobSub subtitle files (IDX/SUB) - Extracted subtitle tracks from MKV files

    Using IDX/SUB extracted from MKV is recommended for easy comparison after OCR.

    Configuring SubExtractor¶

    Options → Files¶

    Configure DGIndex only if using DVD folder sources.
    Note your OCR Data File Location — this stores your character training sessions.

    Options → Subtitles¶

    Uncheck the following:

    Store SRT files as ANSI (instead of UTF-8)

    Then click OK.

    Training the OCR Engine¶

    Training is required for characters SubExtractor does not recognize.
    Once trained for an episode or two of a TV series, the rest should go quickly — assuming consistent fonts.

    SubExtractor provides helpful instructions on the right side of the OCR window.

    Built-in OCR Tips (from SubExtractor)¶

    Things to watch out for:
If a character, like i or j or ? has more than 1 piece to it, select ALL the pieces
(using the mouse or ARROW KEYS) in the top window BEFORE you click on the matching
character below.

If the text in the DVD is in an Italic (slanted) font you must press and hold either
Ctrl key down when selecting your character. You may not care if the resulting final
product has Italics formatting codes in it, but this program can’t find the spaces
between words correctly unless it knows if the characters are in Italics or not,
so please USE THE CTRL KEY FOR ITALICS.

Sometimes characters run together, so a big block of pixels is highlighted in the
top-left window (more than 1 character at once). In this case you need to Split the
block using the ‘Split in 2’ button.

If the patterns in the top-left window look funny: for example, if they are just
big outlines of letters instead of being filled-in, you can hit the ‘Different Palette’
button if it’s enabled. This program tries to find the best combination of colors
to make OCR’ing easy, but it isn’t perfect.

    Additional Practical Tips¶

    • Make sure italics and caps lock are not accidentally active
    • If unsure whether a word is italic: check whether straight lines are slanted
    • Review training under Review and Correct OCR Matches
    • Re-train punctuation (periods, commas, I vs l) frequently — these cause subtle errors
    • Incorrect training? → Delete it → Re-OCR → Re-export

    After OCR: Export and Verify in Subtitle Edit¶

    Open the resulting SRT in Subtitle Edit.

    If you also extracted the original VobSub (.sub):

    • Video → Open video file
    • Video → Open second subtitle track

    Or open the original .sub in a second Subtitle Edit window.

    Spell Check Review¶

    Use this NOT as grammar correction but as an OCR error finder.

    Spell check helps find: - Incorrect characters - Missing or extra spaces - Broken punctuation

    Skip unless it is an actual OCR error.

    If unsure → Cancel → double-click subtitle → Compare to original.

    ⚠ Troubleshooting Common Problems¶

    1. Spacing Issues¶

    Try: - SubExtractor → Advanced Word Spacing Adjustment
    - Manual corrections
    - Find & Replace

    2. Incorrect Italics or merged characters¶

    Means your OCR training is wrong.

    Fix: 1. Go back in SubExtractor → Previous Step
    2. Open Review and Correct OCR Matches
    3. Delete incorrect training
    4. Re-train
    5. Re-export

    3. Fix Common Errors (Subtitle Edit)¶

    Tools → Fix common errors

    Useful for: - Double spaces
    - Noise removal
    - Merged words
    - Incorrect punctuation

    Finishing Up¶

    Most important things to verify:

    • OCR training correctness
    • Periods, commas, ambiguous characters (I vs l)
    • Consistency across multiple samples

    If SubExtractor fails entirely, consider SubRip as a fallback.
